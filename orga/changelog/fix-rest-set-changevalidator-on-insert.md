- FIX REST `/set` endpoint bypassing the `changeValidator` for inserts of new documents. The handler called the validator only on the update branch (when the document already existed on the server), so a client could create new documents that should have been rejected. The handler now also runs the `changeValidator` for inserts (with `assumedMasterState: undefined`, matching the replication `/push` behavior) and returns 403 Forbidden if the validator returns `false`.
