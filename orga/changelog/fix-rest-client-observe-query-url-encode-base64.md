- FIX REST client `observeQuery` not URL-encoding the base64 query string. Standard base64 contains `+` and `/`; in a URL query parameter `+` is decoded by the server as a space, so the server's `atob` rejected the corrupted string with `Invalid character` and the SSE handler crashed silently after the response headers were already sent. Any query whose base64 contained `+` or `/` (for example, queries that filtered by a unicode value such as `firstName: { $eq: '├╗├┐├¥' }`) would never deliver a document and the client just hung. The base64 is now passed through `encodeURIComponent` before being appended to the URL.
